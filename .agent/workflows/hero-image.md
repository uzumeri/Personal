## /hero-image — Generate a Hero Image for a Post

Use this workflow to create or regenerate a hero image for a draft or published post. Use this if the original image generated by `/substack-post` is inappropriate, or if you want to explicitly guide the prompt.

### How to invoke

In chat, say one of these:

> `/hero-image my-post-slug` 
> `/hero-image my-post-slug [optional instructions for the image prompt]`

### What the agent will do

1. **Locate the post** — find the matching `.md` file in `drafts/` or `published/`.
2. **Read the post** — analyze the content to understand the core theme and tone. If the user provided specific instructions, prioritize those.
3. **Generate a Hero Image** — use the `generate_image` tool to create a minimalist, editorial-style illustration capturing the article's theme or following the user's specific prompt.
4. **Save the image** — move the generated image from the artifacts directory to `published/images/YYYY-MM-DD-slug-hero.webp` (or `.png`), overwriting any existing image with the same name.
5. **Update Markdown (if needed)** — ensure the markdown file `published/YYYY-MM-DD-slug-substack.md` (if it exists) has the image tag `![Hero Image - Alt Text](images/YYYY-MM-DD-slug-hero.webp)` placed immediately below the subtitle or title.
6. **Confirm** — display the image and wait for the user's approval. If they reject it, offer to generate another one with tweaked instructions.
